# Warehouse Management System (საწყობის მართვის სისტემა)

საწყობის მართვის სისტემა, რომელიც საშუალებას გაძლევთ მართოთ ნივთები, მათი სტატუსები და ინფორმაცია.

## ფუნქციები

- ✅ ნივთების დამატება (ხელით ან Excel/Word ფაილიდან)
- ✅ ნივთების რედაქტირება
- ✅ ნივთების წაშლა
- ✅ სტატუსების მართვა (შემოსული, გზაშია, საწყობშია, გაცემულია)
- ✅ Excel (.xlsx, .xls) და Word (.docx, .doc) ფაილების იმპორტი
- ✅ სტატისტიკის ჩვენება
- ✅ რესპონსიული დიზაინი (მობილური და დესკტოპი)

## მოთხოვნები

- Node.js 18+ 
- PostgreSQL მონაცემთა ბაზა
- npm ან yarn

## ინსტალაცია

1. **რეპოზიტორის კლონირება:**
```bash
git clone <repository-url>
cd werehouse
```

2. **დამოკიდებულებების დაყენება:**
```bash
npm install
```

3. **გარემოს ცვლადების კონფიგურაცია:**

შექმენით `.env` ფაილი პროექტის root დირექტორიაში:

```env
DATABASE_URL="postgresql://username:password@localhost:5432/database_name"
```

4. **Prisma მიგრაციების გაშვება:**
```bash
npx prisma migrate dev
```

5. **Prisma Client-ის გენერაცია:**
```bash
npx prisma generate
```

## გაშვება

**Development რეჟიმი:**
```bash
npm run dev
```

აპლიკაცია გაიხსნება [http://localhost:3000](http://localhost:3000)

**Production build:**
```bash
npm run build
npm start
```

## გამოყენება

### ხელით ნივთის დამატება

1. გადადით "შემოსული ნივთები" სექციაში
2. დააჭირეთ "ახალი ნივთის დამატება" ღილაკს
3. აირჩიეთ "ხელით შეყვანა" ტაბი
4. შეიყვანეთ მონაცემები:
   - ამანათის ნომერი *
   - სახელი *
   - გვარი *
   - ტელეფონი *
   - ქალაქი *
   - მისამართი *
5. დააჭირეთ "დამატება" ღილაკს

### Excel/Word ფაილიდან იმპორტი

1. გადადით "შემოსული ნივთები" სექციაში
2. დააჭირეთ "ახალი ნივთის დამატება" ღილაკს
3. აირჩიეთ "ფაილის ატვირთვა" ტაბი
4. ატვირთეთ Excel (.xlsx, .xls) ან Word (.docx, .doc) ფაილი
5. სისტემა ავტომატურად გადაიტანს მონაცემებს

### Excel ფაილის ფორმატი

Excel ფაილში უნდა იყოს შემდეგი სვეტები (ქართული ან ინგლისური სახელებით):

| ამანათის ნომერი | სახელი | გვარი | ტელეფონი | ქალაქი | მისამართი |
|-------------------|-----------------|----------------|----------|--------|-----------|
| PRD001            | გიორგი          | ბერიძე        | 555123456| თბილისი| რუსთაველის 10 |

**მხარდაჭერილი სვეტის სახელები:**
- **ამანათის ნომერი**: `ამანათის ნომერი`, `productNumber`, `Product Number`
- **სახელი**: `სახელი`, `კლიენტის სახელი`, `Name`, `First Name`
- **გვარი**: `გვარი`, `კლიენტის გვარი`, `fullName`, `Last Name`
- **ტელეფონი**: `ტელეფონი`, `phone`, `Phone`
- **ქალაქი**: `ქალაქი`, `city`, `City`
- **მისამართი**: `მისამართი`, `address`, `Address`

### Word ფაილის ფორმატი

Word ფაილში მონაცემები უნდა იყოს შემდეგი ფორმატით:

```
ამანათის ნომერი: PRD001
სახელი: გიორგი
გვარი: ბერიძე
ტელეფონი: 555123456
ქალაქი: თბილისი
მისამართი: რუსთაველის 10

[შემდეგი ნივთისთვის ცარიელი ხაზი]
```

### ნივთის რედაქტირება

1. ცხრილში ან კარდში იპოვეთ სასურველი ნივთი
2. დააჭირეთ "რედაქტირება" (მწვანე) ღილაკს
3. შეცვალეთ მონაცემები
4. დააჭირეთ "განახლება" ღილაკს

### ნივთის წაშლა

1. ცხრილში ან კარდში იპოვეთ სასურველი ნივთი
2. დააჭირეთ "×" (წითელი) ღილაკს
3. დაადასტურეთ წაშლა

### სტატუსის შეცვლა

1. ცხრილში ან კარდში იპოვეთ სასურველი ნივთი
2. გამოიყენეთ სტატუსის dropdown მენიუ
3. აირჩიეთ ახალი სტატუსი:
   - **შემოსული** (RECEIVED)
   - **გზაშია** (IN_TRANSIT)
   - **საწყობშია** (IN_WAREHOUSE)
   - **გაცემულია** (RELEASED)

## API Endpoints

### GET /api/items
ყველა ნივთის მიღება

### POST /api/items
ახალი ნივთის შექმნა

**Request Body:**
```json
{
  "productNumber": "PRD001",
  "Name": "გიორგი",
  "fullName": "ბერიძე",
  "phone": "555123456",
  "city": "თბილისი",
  "address": "რუსთაველის 10",
  "status": "RECEIVED"
}
```

### POST /api/items/bulk
მრავალი ნივთის ერთდროულად შექმნა

**Request Body:**
```json
{
  "items": [
    {
      "productNumber": "PRD001",
      "Name": "გიორგი",
      "fullName": "ბერიძე",
      "phone": "555123456",
      "city": "თბილისი",
      "address": "რუსთაველის 10",
      "status": "RECEIVED"
    }
  ]
}
```

### PUT /api/items/[id]
ნივთის განახლება

### DELETE /api/items/[id]
ნივთის წაშლა

## ტექნოლოგიები

- **ORM**: Prisma

## პრობლემების გადაჭრა

### ფაილის იმპორტი არ მუშაობს
- შეამოწმეთ, რომ ფაილის ფორმატი სწორია
- Excel-ისთვის: დარწმუნდით, რომ სვეტების სახელები ემთხვევა მხარდაჭერილ სახელებს

### Database კავშირი
- შეამოწმეთ `.env` ფაილში `DATABASE_URL`
- დარწმუნდით, რომ PostgreSQL მუშაობს
- გაუშვით `npx prisma migrate dev` თუ მიგრაციები არ არის გაშვებული
